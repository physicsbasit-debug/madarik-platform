# منصة مدارك - خطة اختبار RC1

## قبل الاختبار

- تأكد من أن GitHub Actions أخضر.
- استخدم متصفح Chrome أو Edge حديثًا.
- جهّز ثلاثة ملفات:
  1. PDF نصي.
  2. صورة سؤال واضحة.
  3. PDF مصوّر بسيط.

## مسار الاختبار الأساسي

1. افتح التطبيق.
2. أدخل بيانات الورقة.
3. ارفع PDF نصيًا.
4. تحقق من النص المستخرج.
5. حوّل النص إلى بطاقات.
6. ولّد القاموس.
7. نفّذ الترجمة.
8. راجع سؤالًا واحدًا على الأقل.
9. استخدم اعتماد كل الأسئلة النشطة.
10. شغّل فحص الجاهزية.
11. صدّر DOCX.
12. صدّر PDF.
13. نزّل Snapshot JSON.
14. ابدأ مشروعًا جديدًا.
15. استورد Snapshot JSON.
16. تأكد من عودة البيانات.
17. صدّر من المشروع المستورد.

## اختبار الحالات السلبية

- احذف كل الأسئلة وتأكد أن فحص الجاهزية يمنع التصدير.
- ارفع ملفًا غير مدعوم وتأكد من ظهور رسالة مناسبة.
- جرّب صورة ضعيفة الجودة وتأكد أن OCR لا يكسر التطبيق.

## قرار الاختبار

- PASS: صالح للانتقال إلى Phase 2 planning.
- PASS WITH NOTES: يحتاج Fix محدود.
- FAIL: لا يثبت كـRC1.
