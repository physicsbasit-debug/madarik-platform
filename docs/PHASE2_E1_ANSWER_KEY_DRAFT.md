# منصة مدارك - Phase 2-E1 Answer Key Draft

## الغرض

إضافة مسودة نموذج إجابة أولية تساعد المعلم على مراجعة الورقة بعد ترجمتها، دون اعتبارها نموذجًا معتمدًا.

## ما تم

- إضافة نموذج:
  - `AnswerKeyItem`
- إضافة حقل داخل المشروع:
  - `answer_key`
- إضافة خدمة:
  - `backend/app/services/answer_key.py`
- إضافة endpoints:
  - `POST /api/projects/{project_id}/answer-key/draft`
  - `DELETE /api/projects/{project_id}/answer-key`
- عرض مسودة نموذج الإجابة في خطوة التصدير.
- حذف مسودة نموذج الإجابة من الواجهة.

## حدود المرحلة

- المسودة ليست نموذج إجابة نهائيًا.
- لا توجد درجات تفصيلية داخل السؤال.
- لا توجد Rubric متقدمة.
- لا تصدّر المسودة داخل ورقة الطالب.
- لا تستخدم AI خارجيًا لتوليد الإجابات في هذه المرحلة.

## معيار القبول

- توليد مسودة للأسئلة غير المحذوفة.
- تجاهل الأسئلة المحذوفة.
- حفظ `answer_key` داخل المشروع.
- حذف المسودة.
- بقاء اختبارات Backend وFrontend خضراء.
