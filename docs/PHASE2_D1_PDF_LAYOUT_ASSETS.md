# منصة مدارك - Phase 2-D1 PDF Layout & Assets Extraction

## الغرض

إضافة استخراج مبدئي للقطات تخطيط PDF حتى يستطيع المعلم مراجعة الرسوم والجداول بصريًا أثناء الاستخراج.

## ما تم

- إضافة نموذج:
  - `PdfLayoutAssetInfo`
- إضافة حقل داخل المشروع:
  - `layout_assets`
- إضافة خدمة:
  - `backend/app/services/pdf_layout_assets.py`
- إضافة endpoint:
  - `POST /api/projects/{project_id}/layout-assets/pdf`
- إضافة endpoint للحذف:
  - `DELETE /api/projects/{project_id}/layout-assets/{asset_id}`
- عند رفع PDF من الواجهة، يحاول النظام استخراج لقطات تخطيط منخفضة الدقة من أول صفحات PDF.
- عرض لقطات التخطيط في خطوة الاستخراج.
- حذف لقطة تخطيط من الواجهة.

## حدود المرحلة

- لا يوجد ربط ذكي تلقائي بين الرسم والسؤال.
- لا يوجد OCR داخل الصور المستخرجة.
- لا يوجد فصل حقيقي للجداول عن الرسوم.
- لا يتم استخراج كل صفحات PDF الطويلة، بل حد مبدئي آمن.
- الهدف مراجعة بصرية ودعم يدوي، لا فهم PDF كاملًا. هذا سيأتي لاحقًا إن توقفت ملفات PDF عن التصرف كحفريات رقمية.

## معيار القبول

- رفع PDF لا يكسر مسار استخراج النص.
- يتم حفظ `layout_assets` داخل المشروع.
- تظهر اللقطات في الواجهة.
- يمكن حذف لقطة.
- تبقى اختبارات Backend وFrontend خضراء.
