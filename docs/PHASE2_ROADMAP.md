# منصة مدارك - Phase 2 Roadmap

## الغرض

بعد اكتمال Phase 1-RC1، ينتقل المشروع من نموذج تجريبي قابل للاختبار إلى أساس منتج قابل للتوسع. Phase 2 لا تعني إضافة كل شيء دفعة واحدة؛ هذا بالضبط كيف تتحول المشاريع إلى كومة أسلاك سعيدة بنفسها.

## المبادئ الحاكمة

1. لا ميزة كبيرة دون اختبار.
2. لا حفظ دائم قبل حسم نموذج البيانات.
3. لا حسابات قبل وجود تخزين واضح.
4. لا AI خارجي إلزامي قبل طبقة إعدادات وأمان مستقرة.
5. لا استخراج تلقائي للصور والجداول قبل حماية مسار PDF الحالي.
6. كل مرحلة يجب أن تبقى قابلة للرفع والاختبار عبر GitHub Actions.

## التسلسل المقترح

### Phase 2-A0: Roadmap & Scope Gate

مرحلة حالية. توثيق خطة Phase 2 وحصر الأولويات والمخاطر.

### Phase 2-A1: Persistence Foundation

إضافة تخزين دائم للمشاريع بدل الاعتماد على الذاكرة المؤقتة فقط.

الاقتراح:
- SQLite محليًا كبداية تطويرية.
- طبقة Repository تفصل التخزين عن API.
- إبقاء Snapshot JSON كمسار نسخ احتياطي.
- عدم إدخال حسابات بعد.

### Phase 2-A2: Project Library

واجهة لإظهار المشاريع المحفوظة:
- إنشاء مشروع.
- فتح مشروع محفوظ.
- حذف مشروع.
- تكرار مشروع.
- البحث/الفرز البسيط.

### Phase 2-B1: Accounts & Roles

إضافة حسابات وصلاحيات بعد ثبات التخزين:
- مالك/مدير.
- معلم.
- قارئ.
- حماية المشاريع من الخلط بين المستخدمين.

### Phase 2-C1: AI Provider Hardening

تحسين مزود AI الخارجي:
- إعدادات آمنة.
- حدود تكلفة/عدد طلبات.
- سجل ملخص لنتيجة الترجمة.
- fallback واضح.
- اختبارات mock دون مفاتيح حقيقية.

### Phase 2-D1: PDF Layout & Assets Extraction

استخراج مبدئي للصور/الجداول من PDF وربطها بالأسئلة تلقائيًا أو شبه تلقائي.

### Phase 2-E1: Answer Key Drafting

توليد مسودة نموذج إجابة للمعلم فقط، مع تحذيرات مراجعة واضحة.

### Phase 2-F1: Educational Analysis

تحليل تربوي للأسئلة:
- مهارات معرفية.
- أفعال السؤال.
- توزيع الدرجات.
- ملاحظات جودة الورقة.

## المسار الموصى به الآن

ابدأ بـ **Phase 2-A1: Persistence Foundation**.

السبب: الحفظ الدائم هو العمود الفقري قبل الحسابات والمكتبة وAI المتقدم. إضافة حسابات دون تخزين منظم يشبه بناء باب فاخر على خيمة.


## Phase 2-B1: Accounts & Roles Foundation

أضيفت طبقة حسابات وصلاحيات أولية:
- إنشاء حساب المالك عند أول تشغيل.
- تسجيل الدخول والخروج.
- جلسات محفوظة في SQLite.
- أدوار أولية: owner / teacher / reviewer.

الوثيقة التفصيلية:

`docs/PHASE2_B1_ACCOUNTS_ROLES_FOUNDATION.md`


## Phase 2-B2: Project Ownership & Access Rules

أضيف ربط المشاريع بالحساب الحالي وقواعد وصول أولية فوق مكتبة المشاريع.

الوثيقة التفصيلية:

`docs/PHASE2_B2_PROJECT_OWNERSHIP_ACCESS.md`


## Phase 2-B3: User Management Lite

أضيفت إدارة حسابات خفيفة للمالك:
- عرض الحسابات.
- إنشاء حساب.
- تفعيل/تعطيل حساب.
- حماية إدارة الحسابات لتكون للمالك فقط.

الوثيقة التفصيلية:

`docs/PHASE2_B3_USER_MANAGEMENT_LITE.md`


## Phase 2-F1: Educational Analysis Foundation

أضيف تحليل تربوي تأسيسي للورقة:
- حفظ `educational_analysis` داخل المشروع.
- تحليل عدد الأسئلة والدرجات وأوامر السؤال.
- عرض التنبيهات والتوصيات.
- حذف التحليل.

الوثيقة التفصيلية:

`docs/PHASE2_F1_EDUCATIONAL_ANALYSIS_FOUNDATION.md`
