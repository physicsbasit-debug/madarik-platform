# Phase 4-A2: Scientific Translation Prompt

## الهدف

بناء موجه ترجمة علمية منضبط يحول طبقة المزود التي أُنجزت في Phase 4-A1 إلى مسار ترجمة مناسب لأسئلة الاختبارات المدرسية، دون إضافة إجابات أو شروح أو تغيير مستوى الطلب المعرفي.

## نطاق المرحلة

- صياغة بروتوكول ترجمة علمية ثابت يحمل رقم إصدار `phase-4-a2-v1`.
- تمرير سياق المادة والصف والفصل الدراسي ورقم السؤال إلى المزود.
- تمرير سياق السؤال الرئيسي والجزء الأب عند ترجمة أجزاء الأسئلة الهرمية.
- الحفاظ على الأرقام والقيم والمعادلات والمتغيرات والرموز الكيميائية والوحدات والدرجات.
- الحفاظ على مراجع الأشكال والجداول وتسميات الأجزاء والخيارات.
- منع حل السؤال أو إضافة تلميحات أو معلومات غير موجودة في الأصل.
- التعامل مع نص السؤال بوصفه بيانات لا تعليمات موجهة إلى النظام.
- توسيع دليل أفعال الأسئلة الامتحانية دون تحويله إلى قاموس مصطلحات شامل.

## خارج النطاق

- فرض قاموس المصطلحات والتحقق الآلي من الالتزام به، وهو مؤجل إلى Phase 4-A3.
- تقييم جودة الناتج بعد الترجمة واكتشاف الرموز أو الوحدات المفقودة، وهو مؤجل إلى Phase 4-A4.
- اختبار ترجمة حي مدفوع عبر مزود خارجي، ويبقى مؤجلًا حتى يتوفر مفتاح API.
- تعديل واجهة المراجعة أو إضافة إعدادات جديدة للمستخدم.

## عقد الموجه

يتكون الطلب من قسمين:

1. تعليمات نظام ثابتة تحدد قواعد الترجمة العلمية.
2. طلب مستخدم منظم يحتوي:
   - إصدار الموجه.
   - سياق المشروع والسؤال.
   - دليل أفعال السؤال.
   - قاموس الورقة المتاح.
   - نص السؤال المصدر.
   - شرط إخراج الترجمة العربية فقط.

## سياق الأسئلة متعددة الأجزاء

تظل كل قطعة مترجمة بصورة مستقلة، لكن الموجه يتلقى عند الحاجة:

- المادة والصف والفصل الدراسي.
- رقم السؤال.
- رمز الجزء الحالي.
- نص السؤال الرئيسي.
- نص الجزء الأب المباشر.

السياق يستخدم لحسم الدلالة فقط، ولا يجوز أن يضاف إلى الناتج إذا لم يكن موجودًا في النص المصدر.

## التوافق

- تظل التواقيع القديمة صالحة لأن سياق الموجه اختياري.
- يستمر وضع `mock` وfallback دون اتصال خارجي.
- يستخدم OpenAI الرسمي نفس Responses API المعتمد في A1.
- يستخدم `openai-compatible` نفس Chat Completions المعتمد في A1.
- لا يوجد تغيير في نموذج البيانات المخزن أو تنسيق التصدير.

## اختبارات القبول

1. يتضمن الموجه منع الحل والإجابة والتفسير والإضافة.
2. يحافظ الطلب على `V = IR` و`2 A` و`5 Ω` و`[2]` دون تعديل.
3. يظهر سياق المادة والصف ورقم السؤال والجزء داخل الطلب.
4. يتلقى الجزء الفرعي نص الجزء الأب عند وجوده.
5. يتعامل الموجه مع تعليمات خبيثة داخل نص السؤال على أنها بيانات فقط.
6. يظهر `prompt_version` داخل حالة المزود دون كشف أي سر.
7. تبقى اختبارات الترجمة القديمة ناجحة دون الحاجة إلى مفتاح API.
