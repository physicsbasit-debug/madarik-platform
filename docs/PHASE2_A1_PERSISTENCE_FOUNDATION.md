# منصة مدارك - Phase 2-A1 Persistence Foundation

## الغرض

إضافة تخزين دائم للمشاريع باستخدام SQLite مع طبقة Repository، دون إدخال حسابات أو صلاحيات أو مكتبة مشاريع كاملة في هذه المرحلة.

## ما تم اعتماده

- SQLite كبداية محلية بسيطة.
- Repository Layer في:
  - `backend/app/services/project_repository.py`
- استمرار `session_store.py` كطبقة تعامل داخلية مع API.
- حفظ كل مشروع كاملًا كـJSON payload داخل جدول `projects`.
- إضافة فهارس أساسية للترتيب حسب `updated_at`.
- إبقاء Snapshot JSON كما هو.
- عدم إدخال حسابات أو مستخدمين.

## قاعدة البيانات

المسار الافتراضي:

```text
backend/data/madarik.sqlite3
```

يمكن تغييره عبر متغير البيئة:

```text
MADARIK_DB_PATH
```

كما يمكن تغيير مجلد البيانات عبر:

```text
MADARIK_DATA_DIR
```

## لماذا payload JSON الآن؟

هذا اختيار مرحلي مقصود. نماذج Phase 1 لا تزال تتغير، وتفكيك كل جزء إلى جداول كثيرة مبكرًا سيجعل التطوير يشبه تعبئة استمارات بدل بناء منتج.

في Phase لاحقة يمكن ترحيل البيانات إلى جداول أكثر تفصيلًا إذا استقر نموذج البيانات.

## ما لم يدخل

- واجهة مكتبة المشاريع.
- حسابات مستخدمين.
- صلاحيات.
- PostgreSQL.
- تخزين ملفات خارجي.
- Migration system معقد.

## معيار القبول

- إنشاء مشروع يحفظه في SQLite.
- تحديث المشروع يحفظ التغيير.
- مشروع جديد يمكن استرجاعه بعد إنشاء `InMemoryProjectStore` جديد.
- حذف المشروع يحذفه من SQLite.
- Snapshot ما زال يعمل.
- مسار التصدير ما زال يعمل.
