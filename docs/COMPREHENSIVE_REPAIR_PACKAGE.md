# Comprehensive Repair Package

## الهدف

توحيد مراحل منصة مدارك الحديثة مع العقود القديمة المستقرة، وإزالة الأعطال التي
كانت تمنع تشغيل Backend وبناء Frontend أو تسمح بالوصول غير المصرح إلى الموارد.

## الإصلاحات المعتمدة

### التوافق الخلفي لمصادر السحابة

- استعادة عقود Google Drive القديمة دون حذف نموذج المصادر المتعدد.
- فصل `CloudSourceListResponse` القديم عن `CloudSourceRegistryListResponse`.
- الحفاظ على Google Drive وOneDrive داخل المشروع نفسه دون تعارض أسماء.
- تثبيت هوية المصدر عند التحديث وفق `provider + external_id`.

### المسارات وFastAPI

- نقل مسار `GET /projects/{project_id}` إلى نهاية الموجّه.
- منع المسار الديناميكي من حجب المسارات الثابتة مثل:
  - `/projects/cloud-sources`
  - `/projects/differentiated-activities`
  - `/projects/scientific-diagrams`
  - `/projects/assessment-builder`
- إضافة اختبارات فعلية لمسارات التجميع الثابتة.

### الصلاحيات

- التحقق من المالك قبل القراءة والحذف والمعاينة والتصدير والمزامنة.
- إبقاء مالك النظام قادرًا على إدارة جميع الموارد.
- منع حذف النشاط قبل فحص الصلاحية.
- تصفية القوائم بحيث لا تظهر موارد حساب آخر.

### الأنشطة والاختبارات

- إصلاح `DifferentiatedActivityRepository.get/delete` داخل الفئة الصحيحة.
- إصلاح معاينة وتصدير الأنشطة.
- إصلاح وظائف معاينة وتصدير الاختبار التي كانت تشير إلى متغيرات غير موجودة.
- تحديث اختبارات DOCX لتقرأ ملف Word أصليًا بدل معاملته كنص UTF-8.
- جعل فحص عناوين أسئلة PDF متينًا أمام اختلاف استخراج النص العربي.

### Frontend وTypeScript

- إصلاح Imports المكسورة في الملفات الرئيسة.
- استكمال استيراد أنواع الأنشطة والرسوم ومصادر السحابة داخل `api.ts`.
- إصلاح أنواع بناء الاختبار.
- إضافة اختيار الصف والمادة والفصل والوحدة والدرس للأنشطة والرسوم.
- توحيد إصدار `package.json` و`package-lock.json`.

### OneDrive وMicrosoft Graph

- تمثيل روابط المشاركة عبر Graph share token من صيغة `u!`.
- استخدام `/shares/{shareToken}/driveItem` لروابط المشاركة.
- منع استخدام `/me` مع تدفق التطبيق الخلفي Client Credentials.
- دعم العنونة بواسطة `drive_id + item_id` أو `user_id + item_id`.
- استخدام رابط التنزيل المؤقت مباشرة دون تخزينه.
- قبول روابط تنزيل HTTPS صالحة فقط.
- إبقاء الأسرار في متغيرات البيئة وعدم إعادتها للواجهة.

### الأمان والملفات

- تهريب معرفات SVG داخل الخصائص.
- إزالة `dangerouslySetInnerHTML` من معاينة SVG.
- عرض SVG داخل عنصر صورة باستخدام Data URI.
- تنظيف أسماء ملفات DOCX وPDF وPNG وSVG وتنزيلات OneDrive.
- حذف قاعدة SQLite المولدة من حزمة المصدر.
- إزالة إصدارات الاعتمادات المتعارضة.
- دعم غياب مكتبات تشكيل العربية اختياريًا دون منع بدء Backend.

## الفحوص المنفذة

- Python compileall: ناجح.
- Backend full pytest: `500 passed` قبل إضافة اختبار HTTPS، ثم أضيف اختبار إصلاح
  إضافي وتُعاد البوابة النهائية عند تجهيز الحزمة.
- اختبارات الإصلاح الشاملة: تشمل التوافق، ترتيب المسارات، الصلاحيات، OneDrive،
  SVG، أسماء الملفات، المتطلبات، وإصدار Package Lock.
- إنشاء OpenAPI: ناجح، مع 95 عملية فريدة.
- جميع مسارات GET الثابتة: لا توجد استجابة 500.
- TypeScript/TSX parser: ناجح.
- Strict TypeScript audit باستخدام تعريفات مؤقتة للمكتبات الخارجية: ناجح.
- فحص JSON وYAML: ناجح.
- فحص أنماط الأسرار: لم تُكتشف مفاتيح حقيقية.

## قيد بيئة الفحص

تعذر تنفيذ `npm ci` بسبب استجابة `503 Service Temporarily Unavailable` من سجل
npm الداخلي في بيئة الفحص. لذلك لم يُنفذ Vite build أو ESLint الحقيقيان محليًا،
ويجب أن يبقيا ضمن بوابة GitHub Actions بعد رفع الحزمة.

## القرار

هذه الحزمة هي خط الأساس الصحيح قبل Phase 9-C. لا ينبغي متابعة بناء سجل النسخ
واستقبال الملفات السحابية فوق النسخة السابقة المكسورة.
