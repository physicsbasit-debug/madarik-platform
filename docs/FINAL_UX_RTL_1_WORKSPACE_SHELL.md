# Final UX RTL-1: Model 1 Workspace Shell

## القرار

اعتماد نموذج مساحة العمل من ثلاث مراحل بوصفه الهيكل الرسمي لواجهة منصة مدارك، مع اتجاه RTL كامل وشريط مراحل ثابت على يمين الشاشة.

## النطاق

هذه الدفعة تغيّر طبقة العرض والتنقل فقط، ولا تغيّر:

- عقود API.
- نماذج البيانات.
- منطق الترجمة.
- الاستخراج والقص.
- التصدير.
- SQLite وحفظ الجلسات.

## المكونات الجديدة

- `WorkspaceShell.tsx`
- `WorkspaceSidebar.tsx`
- `WorkspaceTopBar.tsx`
- `WorkflowStatusStrip.tsx`

## ما تم تنفيذه

- تخطيط RTL صريح على مستوى مساحة العمل.
- شريط جانبي داكن على يمين الشاشة.
- المراحل الثلاث: البدء والرفع، المراجعة، التصدير.
- شريط علوي مختصر للمشروع وحالة الاتصال والإجراءات العامة.
- مؤشرات حالة مختصرة أسفل الشريط العلوي.
- نقل الحساب والصلاحيات إلى قسم قابل للفتح بدل احتلال مقدمة الصفحة.
- الحفاظ على `StartWorkspace` و`UnifiedReviewWorkspace` و`ExportStep` دون تغيير عقودها.
- تخطيط متجاوب يحول المراحل إلى شريط أفقي على الشاشات الصغيرة.

## بوابة القبول

- Frontend lint ناجح.
- TypeScript build ناجح.
- Vite production build ناجح.
- الاختبارات الثابتة الجديدة ناجحة.
- لا يوجد استيراد لخدمات API داخل مكونات العرض الجديدة.

## خارج النطاق

- إعادة توزيع محتوى شاشة البدء والرفع داخليًا.
- إعادة بناء شاشة المراجعة بثلاثة أعمدة.
- إعادة تصميم شاشة التصدير.
- تحسين أداء القص والترجمة والاستخراج.

تُنفذ هذه العناصر في دفعات مستقلة بعد اختبار الهيكل العام بصريًا ووظيفيًا.
