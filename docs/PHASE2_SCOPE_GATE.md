# منصة مدارك - Phase 2 Scope Gate

## لماذا هذه الوثيقة؟

Phase 1 انتهت بنسخة RC1. قبل إدخال Phase 2 يجب منع توسع غير منضبط. نعم، الأفكار جميلة، لكنها عندما تدخل كلها دفعة واحدة تتحول إلى حديقة حيوانات برمجية.

## مسموح في Phase 2-A

- بناء تخزين دائم.
- إنشاء مكتبة مشاريع محفوظة.
- الحفاظ على Snapshot JSON.
- تحسين الاختبارات.
- تحسين بنية الخدمات دون كسر API الحالي.

## ممنوع في Phase 2-A

- حسابات مستخدمين قبل التخزين.
- AI خارجي إلزامي.
- نموذج إجابة.
- OCR عربي.
- تحليل تربوي.
- إعادة تصميم شاملة للواجهة.
- استخراج تلقائي معقد للصور والجداول.

## شروط بدء Phase 2-A1

- GitHub Actions أخضر على RC1.
- وثائق RC1 موجودة.
- قرار واضح لنوع التخزين الأولي.
- عدم حذف Snapshot JSON.
- اختبار regression كامل يبقى ناجحًا.

## قرار التخزين المقترح

ابدأ بـ SQLite في التطوير المحلي، مع تصميم Repository يسمح لاحقًا بالانتقال إلى PostgreSQL.

السبب:
- بسيط.
- لا يحتاج خدمة خارجية.
- مناسب لاختبار النموذج.
- يمكن اختباره في CI.
- لا يورّطنا في إعدادات استضافة مبكرًا.

## معيار قبول Phase 2-A1

- إنشاء مشروع محفوظ في SQLite.
- فتح مشروع محفوظ.
- تحديث بيانات المشروع.
- حفظ الأسئلة والقاموس والمرفقات.
- بقاء التصدير يعمل.
- بقاء Snapshot يعمل.
- اختبارات Backend وFrontend خضراء.
