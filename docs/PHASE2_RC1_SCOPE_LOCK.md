# Phase 2-RC1 Scope Lock

## القاعدة

Phase 2-RC1 يغلق نطاق المرحلة الثانية. لا تُضاف ميزات جديدة فوقه إلا بعد قرار واضح بفتح Phase 3.

## المسموح بعد RC1

- إصلاح أخطاء واضحة.
- تحسين نصوص توضيحية صغيرة.
- إصلاح اختبارات.
- إصلاح build أو typing.
- توثيق خطوات التشغيل.

## غير المسموح بعد RC1

- إضافة مزود AI جديد.
- إضافة رسوم تفاعلية.
- إضافة تصدير التحليل في PDF/DOCX.
- إضافة مشاركة مشاريع.
- إضافة Rubric تفصيلي.
- تغيير معماري في قاعدة البيانات دون Phase مستقل.

## طريقة التعامل مع الأخطاء

- خطأ واضح ومحدود: `Phase 2-RC1 Fix 1`.
- خطأ متكرر في نفس المسار: مراجعة صغيرة قبل الإصلاح.
- ميزة جديدة: تؤجل إلى Phase 3.

## سبب القفل

النظام أصبح يحتوي طبقات كثيرة: تخزين، حسابات، مشاريع، ترجمة، PDF، OCR، أصول، نموذج إجابة، تحليل، أدوات جودة. إضافة طبقة أخرى الآن بدون تثبيت ستكون مثل بناء طابق جديد فوق بيت لم نختبر أبوابه بعد. مشهد مألوف، ونهايته معروفة.
