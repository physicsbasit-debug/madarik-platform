# قائمة تحقق Final Release Candidate

## 1. المستودع

- [ ] الفرع مبني على أحدث `main`.
- [ ] لا توجد ملفات `__pycache__` أو `.pyc` متتبعة.
- [ ] لا توجد قواعد بيانات أو ZIP أو Patch متتبعة.
- [ ] لا توجد ملفات `.env` أو مفاتيح أو شهادات متتبعة.
- [ ] `git diff --check` ناجح.
- [ ] رقم الإصدار متطابق بين README وFrontend.

## 2. Backend

- [ ] `python -m compileall -q backend/app backend/tests` ناجح.
- [ ] `python -m pip check` ناجح.
- [ ] اختبارات Backend الكاملة ناجحة.
- [ ] PDF النصي يعمل.
- [ ] OCR fallback يعمل.
- [ ] استعادة المشروع من SQLite تعمل.
- [ ] الصلاحيات والملكية تعمل.
- [ ] لا يُعتمد `fallback` كترجمة نهائية.

## 3. Frontend

- [ ] `npm ci` ناجح في بيئة نظيفة.
- [ ] `npm run lint` ناجح.
- [ ] `npm run build` ناجح.
- [ ] استعادة المشروع بعد Refresh تعمل.
- [ ] مراجعة الأسئلة والأجزاء تعمل.
- [ ] مؤشرات مصدر الترجمة وجودة العربية ظاهرة وصحيحة.
- [ ] الواجهة قابلة للاستخدام على الهاتف والكمبيوتر.

## 4. التصدير

- [ ] DOCX يفتح دون إصلاح.
- [ ] PDF يفتح دون أخطاء.
- [ ] RTL صحيح.
- [ ] الأسئلة والأجزاء مرتبة.
- [ ] الدرجات متسقة.
- [ ] الرسوم والجداول موجودة وواضحة.
- [ ] لا تظهر placeholders قديمة.
- [ ] لا تظهر مربعات بدل الرموز.
- [ ] الأسس والوحدات والرموز العلمية سليمة.
- [ ] نسخة الطالب لا تعرض ملاحظات داخلية.

## 5. الترجمة الحية

- [ ] مزود خارجي حقيقي مفعّل.
- [ ] اسم موديل صالح من لوحة المزود.
- [ ] المفتاح محفوظ في متغيرات البيئة فقط.
- [ ] ترجمة ورقة Cambridge كاملة ناجحة.
- [ ] `used_external_provider=true`.
- [ ] لا توجد نتيجة `fallback` معتمدة.
- [ ] لا يوجد نثر إنجليزي غير مفسر.
- [ ] القاموس والرموز العلمية سليمة.
- [ ] محاولة التصحيح الواحدة تعمل عند الحاجة.

## 6. التشغيل والتوثيق

- [ ] README مطابق للحالة.
- [ ] CHANGELOG محدث.
- [ ] GitHub Actions خضراء.
- [ ] Issues القديمة المغلقة وظيفيًا أُغلقت أو وُسمت تاريخية.
- [ ] وثيقة القبول النهائي مملوءة.
- [ ] لا توجد ميزة جديدة دخلت نطاق الإغلاق.

## 7. قرار الإصدار

- [ ] لا توجد عناصر `FIX REQUIRED`.
- [ ] لا توجد عناصر `LIVE BLOCKER`.
- [ ] تم إنشاء Tag فقط بعد القبول.
- [ ] تم إنشاء GitHub Release فقط بعد القبول.
