# Phase 10-A Fix 3: Frontend Lint and Wiring Closure

## سبب الدفعة

وصلت بوابة Phase 10-A إلى Frontend بعد نجاح Backend، ثم أوقفتها ESLint بسبب
9 أخطاء و3 تحذيرات. كانت المشكلات موزعة بين متغيرات ودوال غير مستخدمة،
واعتمادات ناقصة في React Hooks، وخمسة استخدامات صريحة لـ `any`.

## الإصلاحات

### Google Drive

- تحويل دالة التحديث إلى `useCallback` وربط `useEffect` بها.
- إظهار زر اعتماد النسخة عندما تكون حالة المصدر `changed`.
- إبقاء سجل النسخ والحذف والاستيراد في المسار نفسه.

### بنك الأسئلة

- فصل التحميل العام إلى `loadItems` ثابت.
- إبقاء البحث اليدوي مع الفلاتر كما هو.
- تشغيل التحميل الأولي مرة واحدة دون تحذير dependencies ودون بحث تلقائي عند كل تعديل فلتر.
- تثبيت دالة تحديث بنك أسئلة المشروع عبر `useCallback`.

### الصفحة الرئيسية للعلوم

- ربط `onOpenScientificDiagrams` ببطاقة الرسوم والمخططات العلمية.
- ربط `onOpenCloudSources` ببطاقة المصادر السحابية.
- تحديث النص الذي كان يصف OneDrive كمرحلة مستقبلية رغم أن دعمه أصبح موجودًا.

### عقود API

أضيفت واجهات TypeScript صريحة لاستجابات:

- تحقق مخطط الاختبار.
- معاينة ورقة الطالب وأقسامها وأسئلتها.
- معاينة المخطط العلمي وعقده وروابطه.

وبذلك أزيلت استخدامات `any` الخمسة دون تخفيف قواعد ESLint.

## التحقق المحلي

- فحص Syntax لجميع ملفات TypeScript وTSX.
- فحص no-unused على المشروع باستخدام TypeScript مع تعريفات خارجية مؤقتة.
- فحص عدم وجود explicit `any` داخل `frontend/src`.
- اختبارات Backend الكاملة وبوابة Release Audit.

يبقى `npm ci`, `npm run lint`, و`npm run build` للحكم النهائي داخل GitHub Actions،
لأن بيئة البناء المحلية لا تستطيع حل اسم `registry.npmjs.org`.
