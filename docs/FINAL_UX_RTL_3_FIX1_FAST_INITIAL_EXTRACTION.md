# Final UX RTL-3 Fix 1: Fast Initial Extraction

## الهدف

تقليل زمن الانتظار الظاهر بعد رفع الورقة، وفصل القراءة الأولية للنص عن تحليل الرسوم والقص.

## التغيير المعماري المحدود

كان مسار رفع PDF يبقي الواجهة في حالة انشغال حتى ينتهي:

1. استخراج النص أو OCR.
2. استخراج لقطات التخطيط من PDF.

أصبح المسار الآن ينهي القراءة الأولية فور توفر النص، ويؤجل تحليل الرسوم إلى مرحلة المراجعة والأدوات المتقدمة.

## تجربة المستخدم

تظهر مراحل واضحة فور اختيار الملف:

- رفع الملف.
- قراءة PDF.
- OCR عند الحاجة.
- اكتمال القراءة.
- خطأ مع زر إعادة المحاولة.

كما يظهر الزمن المنقضي بالثواني، ويتفعّل زر استخراج الأسئلة فور جاهزية النص.

## نطاق الحماية

لم تتغير:

- عقود API.
- نماذج Backend.
- منطق استخراج النص وOCR.
- منطق الأسئلة والترجمة والتصدير.

## القبول

- استجابة بصرية فورية بعد اختيار الملف.
- عدم تشغيل استخراج التخطيط أثناء القراءة الأولية.
- عرض مرحلة OCR بوضوح.
- عرض الزمن وإعادة المحاولة عند الفشل.
- نجاح lint وTypeScript وVite والاختبارات المركزة.
