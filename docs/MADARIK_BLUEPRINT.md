# منصة مدارك - Blueprint مختصر

منصة مدارك تعالج أوراق الاختبارات الأجنبية وتحولها إلى موارد عربية وثنائية اللغة قابلة للمراجعة والطباعة.

## المسار الحالي

```text
PDF نصي
→ استخراج النص
→ تقسيم الأسئلة
→ توليد قاموس الورقة
→ ترجمة أولية عبر طبقة مزود آمنة
→ مراجعة المعلم
→ تصدير DOCX/PDF
```

## Phase 1-G1

أضافت المرحلة طبقة مزود ترجمة اختيارية دون إجبار المشروع على مفاتيح API. يبقى mock fallback هو الوضع الافتراضي حتى تظل GitHub Actions والتطوير المحلي مستقرة.

## حدود المرحلة

- لا OCR.
- لا نموذج إجابة.
- لا حفظ دائم.
- لا صور أو جداول داخل التصدير.
- لا مفاتيح API داخل الكود أو الواجهة.

---

## تحديث Phase 1-H1

أضيف مسار مرفقات الأسئلة اليدوية: يمكن للمعلم ربط صورة أو جدول ببطاقة سؤال محددة، وتظهر هذه المرفقات في واجهة المراجعة وفي ملفات DOCX/PDF. يبقى استخراج الصور والجداول تلقائيًا من PDF مؤجلًا إلى مرحلة لاحقة.


---

## Phase 1-I2: PDF OCR Fallback

- أضيف دعم رفع الصور PNG/JPG/WEBP لاستخراج النص الإنجليزي مبدئيًا عبر Tesseract OCR.
- بقي PDF النصي مدعومًا كما في Phase 1-C.
- لا يشمل هذا الاستخراج OCR كاملًا لملفات PDF المصوّرة متعددة الصفحات.
- endpoint الجديد: `POST /api/projects/{project_id}/upload-image-ocr`.
- النص المستخرج من الصورة يستخدم في مسار تقسيم الأسئلة الحالي.


## Phase 1-J1: Export Readiness Guard

أضيف فحص جاهزية قبل التصدير عبر endpoint:

`GET /api/projects/{project_id}/readiness`

يفحص وجود أسئلة قابلة للتصدير، وجود ترجمة، حالات تحتاج مراجعة، الدرجات، والأسئلة المحذوفة. يمنع التصدير عند وجود موانع واضحة مثل عدم وجود أسئلة أو حذف كل الأسئلة.
