# Phase 9-C: Cloud Source Refresh, Version History, and Project Intake

## الهدف

إغلاق مسار مصادر السحابة بسجل نسخ دائم، واعتماد يدوي، وإدخال النسخة
المعتمدة إلى مشروع مدارك.

## سجل النسخ

كل نسخة تحفظ:

- ETag.
- بصمة محتوى مستقرة.
- SHA-256 عند تنزيل الملف.
- الحجم.
- تاريخ التعديل الخارجي.
- المسار المحلي.
- حالة النسخة: detected أو accepted أو superseded.

النسخة الأولى تعتمد تلقائيًا بوصفها خط الأساس. أي نسخة لاحقة تبقى بانتظار
الاعتماد، ولا تستبدل النسخة المعتمدة بصمت.

## إدخال المشروع

- يقبل فقط نسخة معتمدة.
- يتطلب ملفًا مُنزّلًا موجودًا فعليًا.
- يدعم PDF في هذه المرحلة.
- يستخدم نفس مسار استخراج النص وتقرير قبول الورقة الكامل.
- يحدث المشروع الحالي أو ينشئ مشروعًا جديدًا عند عدم تحديد مشروع.

## السلامة

- التحقق من ملكية المصدر قبل قراءة النسخ أو اعتمادها أو إدخالها.
- التحقق أن النسخة تنتمي إلى المصدر.
- حذف سجل النسخ عند حذف المصدر.
- عدم تخزين رابط التنزيل المؤقت في سجل النسخة.

## خارج النطاق

- OCR تلقائي للنسخة السحابية المصورة.
- إدخال صور منفردة.
- مزامنة مجلد كامل.
- تشغيل تحديثات دورية في الخلفية.
