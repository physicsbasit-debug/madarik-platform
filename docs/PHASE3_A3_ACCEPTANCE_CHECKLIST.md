# Phase 3-A3 Acceptance Checklist

## التحليل ونموذج البيانات

- [x] اكتشاف الأجزاء الأبجدية والرومانية والرقمية.
- [x] الاحتفاظ بالأب الهيكلي الفارغ.
- [x] ربط الأجزاء الفرعية بالأب عبر `parent_id`.
- [x] عدم تحويل خيارات MCQ إلى أجزاء سؤال.
- [x] توافق المشاريع والأسئلة القديمة بلا `parts` أو بلا `parent_id`.

## المراجعة والحفظ

- [x] عرض الأجزاء الرئيسية والفرعية داخل شاشة المراجعة.
- [x] تعديل الوسم والنص والترجمة والدرجة.
- [x] إضافة جزء رئيسي وفرعي.
- [x] تغيير تبعية الجزء.
- [x] تحريك الجزء داخل إخوته وحفظ `order_index`.
- [x] استمرار الترتيب والتبعية بعد `Ctrl + Shift + R`.
- [x] حذف الأب مع ترقية الأبناء بدل حذفهم.
- [x] حفظ البيانات في Backend وSQLite.

## الترجمة

- [x] ترجمة كل جزء ذي نص بصورة مستقلة.
- [x] الحفاظ على الأب الهيكلي الفارغ دون نص بديل مصطنع.
- [x] الحفاظ على `parent_id` والترتيب بعد الترجمة.
- [x] استمرار ترجمات الأجزاء بعد تحديث الصفحة.

## الدرجات والجاهزية

- [x] حساب المجموع الهرمي دون مضاعفة الأب والأبناء.
- [x] إظهار تحذير إرشادي عند اختلاف الدرجة العامة عن مجموع الأجزاء.
- [x] استمرار التصدير عند وجود التحذير.
- [x] اعتماد مجموع الأجزاء بإجراء صريح من المستخدم.
- [x] اختفاء التعارض بعد الاعتماد والحفظ.
- [x] استخدام مجموع الأجزاء عند غياب الدرجة العامة.

## Word وPDF

- [x] تصدير الأب والأبناء بترتيب هرمي.
- [x] إزاحة الأجزاء الفرعية بصريًا.
- [x] إظهار الوسم والدرجة لكل جزء.
- [x] عدم تكرار النص الكامل للسؤال عند وجود أجزاء منظمة.
- [x] عدم طباعة نص fallback للأب الهيكلي الفارغ.
- [x] سلامة اتجاه ومحاذاة عناوين Word.
- [x] استمرار المرفقات بعد كتلة الأجزاء.

## أوامر القبول

```bash
cd backend
source .venv/bin/activate
pytest -q

cd ../frontend
nvm use 22
npm run lint
npm run build

cd ..
git diff --check
git status -sb
```

لا تغلق المرحلة إذا فشل أي اختبار آلي أو عاد خلل وظيفي في الحفظ أو الحركة أو التصدير.
