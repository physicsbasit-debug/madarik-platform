# Phase 5-C: Reuse Question Bank Items in Projects

## الهدف

إعادة استخدام سؤال محفوظ داخل المشروع الحالي مع إنشاء نسخة مستقلة وآمنة.

## عملية النسخ

تُنشأ نسخة جديدة تحتوي على:

- معرف سؤال جديد.
- معرفات جديدة للأجزاء.
- معرفات جديدة للمرفقات المضمنة.
- ترتيب جديد في نهاية أسئلة المشروع.
- حالة `needs_review`.
- مرجع إلى عنصر بنك الأسئلة والمشروع المصدر.
- ملاحظة مراجعة واضحة.

## البيانات المحفوظة

- النص الأصلي والمترجم.
- الخيارات والأجزاء.
- الدرجات.
- التصنيف المعرفي.
- الارتباط المنهجي ونواتج التعلم.
- المرفقات المضمنة.

## البيانات التي تُمسح

تُمسح البيانات الخاصة بسياق المشروع المصدر:

- `linked_layout_asset_ids`
- أرقام صفحات المصدر.
- بداية ونهاية صفحات المصدر.

السبب أن أصول PDF وتخطيط صفحاته لا توجد بالضرورة داخل المشروع الهدف.

## منع التكرار

لا يمكن إدخال عنصر البنك نفسه مرتين في المشروع نفسه. تعيد العملية السؤال الموجود بدل إنشاء نسخة أخرى.

## خارج النطاق

- اختيار مشروع هدف مختلف من الواجهة.
- الاستيراد الجماعي.
- مزامنة نسخة البنك مع النسخة المعاد استخدامها.
- إنشاء اختبار مباشرة.
