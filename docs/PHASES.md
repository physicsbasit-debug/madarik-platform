# خطة مراحل منصة مدارك

## Phase 0 - Skeleton

الحالة: مكتملة.

الأهداف:

- إنشاء هيكل `frontend/`, `backend/`, `docs/`.
- إعداد React + TypeScript + Vite.
- إعداد FastAPI.
- إعداد GitHub Actions.
- اختبار Backend وFrontend build.

## Phase 1-A - Static Multi-Step UI

الحالة: مكتملة عمليًا بعد إصلاح GitHub Actions.

الأهداف:

- واجهة RTL متعددة الخطوات.
- بيانات ورقة قابلة للتعديل داخل الواجهة.
- رفع ملف شكلي.
- أسئلة تجريبية.
- قاموس تجريبي.
- بطاقات مراجعة.
- حذف/استعادة/ترتيب الأسئلة.
- شاشة تصدير شكلية.

## Phase 1-B - Backend API Integration

الحالة: هذه الحزمة.

الأهداف:

- إنشاء جلسة مشروع من الواجهة عبر FastAPI.
- تحميل بيانات تجريبية من Backend.
- مزامنة بيانات الورقة.
- مزامنة معلومات الملف شكليًا.
- مزامنة تعديل الأسئلة.
- مزامنة ترتيب الأسئلة.
- مزامنة قاموس الورقة.
- عرض حالة الاتصال بالخلفية.

ما لا يدخل:

- AI.
- OCR.
- قراءة PDF حقيقية.
- تصدير DOCX/PDF حقيقي.

## Phase 1-C - PDF Text Extraction

مؤجلة.

الأهداف المتوقعة:

- دعم PDF نصي فقط أولًا.
- استخراج النص الخام.
- رسائل خطأ واضحة عند عدم وجود نص.
- عدم إدخال OCR بعد.

## Phase 1-D - Question Parser

مؤجلة.

الأهداف المتوقعة:

- تقسيم النص إلى أسئلة.
- التقاط أرقام الأسئلة.
- التقاط الدرجات إن أمكن.
- إنشاء بطاقات أسئلة من النص الحقيقي.

## Phase 1-E - Glossary + Translation

مؤجلة.

الأهداف المتوقعة:

- قاموس مصطلحات أولي.
- ترجمة سؤالًا سؤالًا.
- استخدام قاموس أوامر السؤال.
- مراجعة الترجمة.

## Phase 1-F - DOCX/PDF Export

مؤجلة.

الأهداف المتوقعة:

- تصدير DOCX RTL.
- تصدير PDF RTL.
- رأس ورقة رسمي.
- شعار اختياري.
- نسخة عربية ونُسخة ثنائية اللغة.
