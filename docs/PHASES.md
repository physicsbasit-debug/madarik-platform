# خطة مراحل منصة مدارك

## Phase 0 - Skeleton

الحالة: مكتملة.

الأهداف:

- إنشاء هيكل `frontend/`, `backend/`, `docs/`.
- إعداد React + TypeScript + Vite.
- إعداد FastAPI.
- إعداد GitHub Actions.
- اختبار Backend وFrontend build.

## Phase 1-A - Static Multi-Step UI

الحالة: مكتملة.

الأهداف:

- واجهة RTL متعددة الخطوات.
- بيانات ورقة قابلة للتعديل داخل الواجهة.
- رفع ملف شكلي.
- أسئلة تجريبية.
- قاموس تجريبي.
- بطاقات مراجعة.
- حذف/استعادة/ترتيب الأسئلة.
- شاشة تصدير شكلية.

## Phase 1-B - Backend API Integration

الحالة: مكتملة.

الأهداف:

- إنشاء جلسة مشروع من الواجهة عبر FastAPI.
- تحميل بيانات تجريبية من Backend.
- مزامنة بيانات الورقة.
- مزامنة معلومات الملف شكليًا.
- مزامنة تعديل الأسئلة.
- مزامنة ترتيب الأسئلة.
- مزامنة قاموس الورقة.
- عرض حالة الاتصال بالخلفية.

## Phase 1-C - PDF Text Extraction

الحالة: مكتملة.

الأهداف:

- دعم رفع PDF نصي حقيقي.
- استخراج النص الخام من PDF القابل للتحديد.
- حفظ نتيجة الاستخراج داخل جلسة المشروع.
- عرض مقتطف النص المستخرج في الواجهة.
- رسالة واضحة إذا كان الملف يحتاج OCR.

ما لا يدخل:

- OCR.
- تقسيم الأسئلة الحقيقي.
- AI.
- الترجمة.
- التصدير DOCX/PDF.

## Phase 1-D - Question Parser

الحالة: هذه الحزمة.

الأهداف:

- تقسيم النص المستخرج إلى أسئلة مستقلة.
- التقاط أرقام الأسئلة.
- التقاط الدرجات إن أمكن.
- إنشاء بطاقات `QuestionItem` من النص الحقيقي.
- إظهار الأسئلة الناتجة في شاشة المراجعة.

ما لا يدخل:

- OCR.
- ترجمة AI.
- استخراج مصطلحات حقيقي.
- ربط الصور والجداول بالسؤال.
- التصدير DOCX/PDF.

## Phase 1-E - Glossary + Translation

مؤجلة.

- ترجمة سؤالًا سؤالًا.
