# مراحل بناء منصة مدارك

## Phase 0: Project Skeleton

الهدف: إنشاء هيكل مشروع نظيف للواجهة والخلفية والتوثيق.

معايير القبول:

- وجود frontend يعمل.
- وجود backend يعمل.
- وجود docs أساسية.
- لا توجد ترجمة أو OCR أو تصدير فعلي.

## Phase 1-A: Static Multi-step UI

الهدف: بناء واجهة ثابتة متعددة الخطوات ببيانات تجريبية.

معايير القبول:

- التنقل بين الخطوات يعمل.
- حالة المشروع لا تضيع أثناء التنقل.
- بطاقات أسئلة تجريبية تظهر.
- تعديل، حذف، ترتيب الأسئلة التجريبية يعمل.
- RTL سليم.

## Phase 1-B: Initial Backend Integration

الهدف: ربط الواجهة بالخلفية وإنشاء مشروع ورفع ملف تجريبي.

## Phase 1-C: Text Extraction

الهدف: استخراج نص PDF نصي أولًا.

## Phase 1-D: Question Parser

الهدف: تقسيم النص إلى أسئلة مستقلة.

## Phase 1-E: Glossary + Translation

الهدف: قاموس الورقة والترجمة سؤالًا سؤالًا.

## Phase 1-F: DOCX/PDF Export

الهدف: تصدير Word وPDF بتنسيق RTL.
