# مراحل بناء منصة مدارك

## Phase 0 - Project Skeleton

الحالة: مكتملة ومعتمدة.

تم فيها إنشاء الهيكل الأساسي:

- frontend
- backend
- docs
- GitHub Actions
- README

## Phase 1-A - Static Multi-Step UI

الحالة: جاهزة للرفع والاختبار.

هدف المرحلة:

- بناء واجهة ثابتة متعددة الخطوات.
- اختبار تجربة المستخدم قبل إدخال المعالجة الحقيقية.
- حفظ حالة المشروع داخل الواجهة أثناء التنقل.
- مراجعة أسئلة تجريبية عبر بطاقات.

المتضمن:

- بيانات الورقة.
- رفع ملف شكلي.
- استخراج أسئلة تجريبية.
- قاموس مصطلحات تجريبي.
- مراجعة الأسئلة.
- تعديل الترجمة والدرجات.
- حذف واستعادة وترتيب.
- شاشة تصدير شكلية.

غير المتضمن:

- OCR.
- AI.
- PDF extraction.
- DOCX/PDF export.
- Backend integration.

معيار القبول:

- Frontend build ينجح.
- Backend tests تنجح.
- GitHub Actions أخضر.
- لا خلط مع وظائف المراحل اللاحقة.
