# سجل قرارات منصة مدارك

## القرارات المثبتة

- الاسم: منصة مدارك.
- البداية بمواد العلوم والفيزياء والكيمياء والأحياء، دون حصر الاسم في العلوم.
- الواجهة: React + TypeScript.
- الخلفية: Python + FastAPI.
- Phase 1 تعتمد واجهة RTL متعددة الخطوات.
- لا حسابات دخول في البداية.
- لا حفظ دائم للمشاريع في Phase 1.
- حفظ مؤقت للجلسة فقط.
- التصدير النهائي المستهدف لاحقًا: DOCX + PDF.
- النسخ النهائية في Phase 1 لاحقًا: عربية نظيفة وثنائية اللغة نظيفة.
- نموذج الإجابة والنسخة التدريبية المشروحة مؤجلان.
- AI وOCR مؤجلان حتى تثبيت أساس القراءة والربط.

## قرارات Phase 1-B

- الواجهة تتصل بـ FastAPI.
- Backend يعطي بيانات تجريبية مملوكة للخلفية.
- معلومات الملف فقط تُرسل إلى Backend، لا الملف الحقيقي.
- كل تعديل في السؤال أو القاموس أو الترتيب يُزامن مع Backend مؤقتًا.
- عند فشل الاتصال، تعمل الواجهة محليًا ببيانات fallback.
- GitHub Actions يستخدم Node 22 وnpm registry العامة لتجنب مشاكل تثبيت الاعتمادات.

## قرارات Phase 1-C

- يتم رفع ملف PDF حقيقي في هذه المرحلة عبر `multipart/form-data`.
- الاستخراج يدعم PDF النصي فقط، أي النص القابل للتحديد.
- لا يتم تشغيل OCR في Phase 1-C.
- إذا كان PDF مصوّرًا أو بلا نص، تظهر رسالة واضحة بأن OCR مؤجل.
- لا يتم تقسيم الأسئلة من النص الحقيقي في هذه المرحلة؛ ذلك مؤجل إلى Phase 1-D.
- تبقى الأسئلة والقاموس التجريبيان موجودين لاختبار واجهة المراجعة فقط.
- تُحفظ نتيجة الاستخراج داخل جلسة المشروع المؤقتة، ولا يوجد حفظ دائم.
