# Final UX RTL-3: Three-Column Review Workspace

## الهدف

تحويل شاشة المراجعة إلى مساحة عمل RTL مركزة حول سؤال واحد، دون تغيير عقود API أو منطق الترجمة والربط والحفظ.

## التوزيع

من اليمين إلى اليسار:

1. قائمة الأسئلة وحالاتها.
2. السؤال المحدد والترجمة والأجزاء والإجراءات.
3. الرسم أو الجدول المرتبط بالسؤال.

## السلوك

- اختيار سؤال من القائمة يعرض بطاقته فقط في مساحة العمل.
- تبقى إعادة الترجمة والحفظ والاعتماد والحذف والترتيب مرتبطة بالعقود الحالية.
- تعرض المراجع البصرية المرتبطة في عمود مستقل.
- يبقى معرض اللقطات الخام والأدوات التفصيلية ضمن الأدوات المتقدمة.
- على الشاشات المتوسطة ينتقل عمود الرسم أسفل مساحة المراجعة.
- على الهاتف تتحول الأعمدة إلى تسلسل رأسي.

## حدود المرحلة

لم تغيّر هذه المرحلة:

- API endpoints.
- نماذج البيانات.
- منطق الترجمة.
- منطق قص الصور.
- منطق الربط التلقائي.
- منطق التصدير.
