# منصة مدارك - Phase 2-B2 Project Ownership & Access Rules

## الغرض

ربط المشاريع الجديدة والمستوردة بالحساب الحالي عند وجود جلسة دخول، وإضافة قواعد وصول أولية فوق مكتبة المشاريع.

## ما تم

- إضافة الحقل `owner_account_id` إلى `ProjectSession`.
- إضافة العمود `owner_account_id` إلى جدول `projects` مع ترقية تلقائية للجداول القديمة.
- ربط المشروع الجديد بالحساب الحالي إذا أرسل الطلب Bearer token.
- ربط Snapshot المستورد بالحساب الحالي إذا كان المستخدم مسجل الدخول.
- تصفية مكتبة المشاريع:
  - المالك `owner` يرى كل المشاريع.
  - الأدوار الأخرى ترى مشاريعها والمشاريع القديمة غير المرتبطة بحساب.
  - غير المسجل يرى المشاريع غير المحمية فقط.
- منع الوصول إلى مشروع مملوك لحساب آخر عند وجود قواعد الملكية.
- إرسال Authorization تلقائيًا من الواجهة إلى API بعد تسجيل الدخول.

## ما لم يدخل

- إدارة مستخدمين كاملة.
- نقل ملكية مشروع.
- مشاركة مشروع مع مستخدم آخر.
- صلاحيات تفصيلية داخل كل تبويب.
- أدوار مدرسية متقدمة.

## ملاحظة

هذه قواعد وصول أولية. المرحلة التالية يمكن أن توسعها إلى إدارة مستخدمين ومشاركة، لكن هذه المرحلة تكفي لجعل المشاريع لا تعيش بلا صاحب مثل ملف اسمه final_final_really_final.pdf.
