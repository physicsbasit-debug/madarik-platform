# منصة مدارك - بروتوكول اختبار قبول Phase 1-N2

## الغرض

اختبار النسخة التجريبية داخليًا قبل تثبيت RC1. هذه القائمة لا تختبر جمال النوايا، بل تختبر أن دورة العمل تعمل فعليًا من ملف مرفوع إلى تصدير نهائي.

## ملفات الاختبار المقترحة

استخدم ثلاثة أنواع من الملفات:

1. PDF نصي واضح من 2 إلى 5 أسئلة.
2. صورة واضحة لسؤال أو سؤالين بصيغة PNG/JPG/WEBP.
3. PDF مصوّر بسيط من صفحة أو صفحتين.

## خطوات الاختبار اليدوي

### 1. إنشاء مشروع

- افتح المنصة.
- أدخل بيانات المدرسة والمادة والصف والفصل.
- اختر نوع النسخة: عربية.
- ارفع شعارًا اختياريًا.

### 2. رفع واستخراج

- ارفع PDF نصيًا.
- تأكد من ظهور النص المستخرج.
- حوّل النص إلى بطاقات أسئلة.

### 3. القاموس والترجمة

- ولّد القاموس.
- عدّل مصطلحًا واحدًا.
- نفّذ ترجمة الأسئلة.

### 4. المراجعة

- عدّل ترجمة سؤال.
- غيّر درجة سؤال.
- احذف سؤالًا ثم استعده.
- أضف مرفق صورة/جدول لسؤال.
- استخدم زر اعتماد كل الأسئلة النشطة.

### 5. الجاهزية والتصدير

- شغّل فحص الجاهزية.
- صدّر DOCX.
- صدّر PDF.
- افتح الملفين وتحقق من اتجاه الكتابة RTL ومن ظهور الأسئلة.

### 6. حفظ العمل

- نزّل نسخة JSON.
- ابدأ مشروعًا جديدًا.
- استورد نسخة JSON.
- تحقق من عودة البيانات والأسئلة والقاموس والمرفقات.

## معيار النجاح

تُعد النسخة جاهزة لـ RC1 إذا:

- نجح GitHub Actions.
- نجحت دورة PDF النصي كاملة.
- نجح تصدير DOCX وPDF وفتحا بصريًا.
- نجح حفظ واستيراد JSON.
- لم يظهر كسر واضح في RTL أو أزرار الخطوات.


## Phase 1-RC1: Release Candidate

تم تثبيت نسخة مرشحة أولى بعد اكتمال مسار Phase 1 الأساسي واختبارات القبول الآلية. أي إضافات كبيرة تؤجل إلى Phase 2.
