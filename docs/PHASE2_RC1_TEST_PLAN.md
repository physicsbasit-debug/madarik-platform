# Phase 2-RC1 Test Plan

## 1. اختبارات آلية

من جذر المستودع:

```bash
cd backend
pytest -q
```

ومن الواجهة:

```bash
cd frontend
npm install --no-audit --no-fund
npm run build
```

## 2. اختبار Smoke يدوي

1. افتح التطبيق.
2. أنشئ حساب المالك إذا لم يكن موجودًا.
3. سجّل الدخول.
4. أنشئ مشروعًا جديدًا.
5. ارفع PDF نصيًا بسيطًا.
6. حوّل النص إلى بطاقات أسئلة.
7. ولّد القاموس.
8. ولّد الترجمة.
9. راجع بطاقة واحدة واعتمدها.
10. ولّد مسودة نموذج الإجابة.
11. ولّد التحليل التربوي.
12. ولّد أدوات الجودة.
13. شغّل فحص الجاهزية.
14. صدّر DOCX.
15. صدّر PDF.
16. ارجع إلى مكتبة المشاريع وافتح المشروع مرة أخرى.

## 3. اختبار الصلاحيات

1. أنشئ حساب معلم.
2. سجّل الدخول بالمعلم.
3. تأكد أن إدارة الحسابات لا تظهر للمعلم.
4. تأكد أن المستخدم لا يستطيع فتح مشروع مملوك لحساب آخر.
5. تأكد أن المالك يستطيع رؤية كل المشاريع.

## 4. اختبار البيانات

تأكد من عدم رفع ملفات قاعدة البيانات إلى GitHub:

```text
backend/data/*.sqlite3
backend/data/*.sqlite3-*
```
