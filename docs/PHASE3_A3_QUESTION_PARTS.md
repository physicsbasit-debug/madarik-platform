# Phase 3-A3: Structured Question Parts

## الهدف

تحويل الأسئلة المركبة من نص واحد صعب المراجعة إلى أجزاء منظمة قابلة للتحرير والترجمة والحفظ والتصدير، مع دعم العلاقة الهرمية بين الجزء الرئيسي وأجزائه الفرعية.

مثال البنية المعتمدة:

```text
(d)
(e) [3]
  (i) [1]
  (ii) [2]
```

## نموذج البيانات

يحفظ كل `QuestionPart` الحقول التالية:

- `id`
- `label`
- `original_text`
- `translated_text`
- `marks`
- `parent_id`
- `order_index`

تبقى المشاريع القديمة متوافقة؛ فإذا غاب `parts` تكون القائمة فارغة، وإذا غاب `parent_id` يعامل الجزء بوصفه جزءًا رئيسيًا.

## المراحل المنجزة

### Phase 3-A3a: Backend Foundation

- تحليل علامات الأجزاء الأبجدية والرومانية والرقمية بصورة محافظة.
- عدم تحويل خيارات MCQ الكبيرة `A-D` إلى أجزاء سؤال.
- الحفاظ على النص الأصلي ومراجع الأشكال والوحدات.
- إضافة `parts` إلى نموذج السؤال والحفظ الدائم.

### Phase 3-A3b: Review UI

- عرض الأجزاء داخل شاشة المراجعة.
- تعديل الوسم والنص الأصلي والترجمة والدرجة.
- إضافة الجزء وحذفه وتحريكه.
- حفظ التعديلات عبر API وSQLite واستعادتها بعد تحديث الصفحة.

### Phase 3-A3c: Independent Translation and Structured Export

- ترجمة كل جزء بصورة مستقلة.
- حفظ ترجمات الأجزاء.
- تصدير الأجزاء بصورة منظمة إلى Word وPDF.
- منع تكرار النص الكامل عند وجود أجزاء منظمة.
- تثبيت اتجاه ومحاذاة عناوين Word.

### Phase 3-A3d: Hierarchical Question Parts

- إضافة `parent_id`.
- الاحتفاظ بالأب الهيكلي الفارغ مثل `(e)`.
- ربط `(i)` و`(ii)` بالأب المناسب.
- إضافة جزء فرعي وتغيير التبعية.
- تحريك الأجزاء داخل مستوى الإخوة مع حفظ `order_index`.
- ترقية الأبناء إلى أجزاء رئيسية عند حذف الأب.
- الحفاظ على الهرمية في الترجمة وWord وPDF.

### Phase 3-A3e: Marks Consistency Guidance

- حساب مجموع الأجزاء دون مضاعفة درجة الأب مع درجات أبنائه.
- إظهار تنبيه إرشادي عند اختلاف درجة السؤال العامة عن المجموع البنيوي.
- إبقاء التصدير متاحًا مع التنبيه.
- إضافة إجراء صريح لاعتماد مجموع الأجزاء بوصفه درجة السؤال.
- استخدام مجموع الأجزاء عند غياب الدرجة العامة.

### Phase 3-A3f: Regression and Acceptance Lock

- إضافة اختبار قبول شامل لمسار السؤال الهرمي.
- تثبيت التحليل والتوافق القديم والحفظ والترجمة واتساق الدرجات والتصدير.
- حماية ترتيب الإخوة وعلاقة الأب والابن عبر snapshot/API round trip.
- توثيق قائمة القبول النهائية للعنقود A3.

## قواعد الدرجات

- درجة السؤال العامة هي الدرجة المستخدمة في رأس السؤال والتجميع إذا كانت محددة.
- عند غياب الدرجة العامة، يستخدم المجموع البنيوي للأجزاء.
- إذا كانت درجة الأب تلخص أبناءه، لا تجمع درجة الأب مرة ثانية مع درجات الأبناء.
- اختلاف الدرجة العامة عن مجموع الأجزاء يولد تحذيرًا إرشاديًا ولا يمنع التصدير.

## التوافق والاستثناءات

- الأسئلة العادية والأسئلة القديمة بلا أجزاء لا تتأثر.
- خيارات MCQ لا تعامل كأجزاء داخلية.
- النص المشوه الناتج من OCR وجودة مزود الترجمة ليسا ضمن مسؤولية بنية الأجزاء.
- الترجمة النهائية تظل بحاجة إلى مراجعة المعلم.

## نقطة الإغلاق

- تصدير Word وPDF للبنية الهرمية.
- اختبار الانحدار الشامل في `test_phase3_a3_multipart_regression.py`.
