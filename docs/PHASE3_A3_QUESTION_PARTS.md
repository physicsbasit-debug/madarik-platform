# Phase 3-A3a: Question Parts Backend Foundation

## الهدف

إضافة تمثيل منظم للأجزاء الداخلية للأسئلة المركبة، مثل:

- `(a)` و`(b)`
- `(i)` و`(ii)` و`(iii)`
- الأجزاء الرقمية مثل `(1)` و`(2)`

## القرارات

- يظل `QuestionItem.original_text` محفوظًا كما هو.
- يضاف الحقل `parts` بصورة توافقية.
- القيمة الافتراضية للمشاريع القديمة قائمة فارغة.
- لا تتغير الواجهة أو الترجمة أو التصدير في هذه الدفعة.
- خيارات MCQ الكبيرة `A-D` لا تعامل كأجزاء سؤال.
- مراجع الأشكال والوحدات العلمية تبقى دون تغيير.
- لا يخترع المحلل أجزاء عند غياب علامات واضحة.

## نموذج الجزء

كل جزء يحتوي على:

- `id`
- `label`
- `original_text`
- `translated_text`
- `marks`
- `order_index`

## مؤجل للمراحل التالية

- واجهة مراجعة الأجزاء.
- تعديل الأجزاء عبر API.
- ترجمة كل جزء على حدة.
- تصدير الأجزاء المنظمة.

تنتقل هذه البنود إلى Phase 3-A3b وPhase 3-A3c.
