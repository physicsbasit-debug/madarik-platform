# Final UX RTL-2: Start & Upload Workspace

## الهدف

إعادة ترتيب شاشة **البدء والرفع** داخل الهيكل RTL المعتمد، دون تغيير عقود
Backend أو منطق إدارة المشاريع والرفع والاستخراج.

## التوزيع المعتمد

تقرأ الشاشة من اليمين إلى اليسار:

1. بيانات الورقة.
2. رفع الملف وحالته.
3. مكتبة المشاريع.

ويظهر أسفلها ملخص معالجة واحد وزر رئيسي لاستخراج الأسئلة.

## ما تغير

- إزالة المقدمة الكبيرة والمتكررة.
- توحيد بيانات الورقة داخل بطاقة مدمجة.
- إبقاء البيانات الثانوية والشعار وتفضيلات التصدير داخل قسم اختياري مطوي.
- تحويل رفع الملف إلى منطقة سحب واختيار واضحة.
- عرض حالة الملف والنص والعناصر البصرية بجوار الرفع.
- ضغط مكتبة المشاريع لتناسب العمود الثالث مع بقاء:
  - تحديد الكل.
  - الحذف الجماعي.
  - التحديث.
  - الفتح والحذف الفردي.
- إنشاء بطاقة واحدة لملخص المعالجة والإجراء التالي.
- تحسين الاستجابة للشاشات المتوسطة والهاتف.

## الحماية المعمارية

لم تتغير:

- خدمات API.
- نماذج البيانات.
- callbacks في `App.tsx`.
- منطق رفع الملف.
- منطق استخراج النص والأسئلة.
- منطق حذف وفتح المشاريع.
- الترجمة أو التصدير.

## معيار القبول

- تظهر بيانات الورقة يمينًا، والرفع في الوسط، والمشاريع يسارًا.
- لا تضيع أي حقول أو تفضيلات موجودة سابقًا.
- تبقى إجراءات المشاريع والرفع والاستخراج مربوطة بالـcallbacks الأصلية.
- تتحول الشاشة إلى عمود واحد على الهاتف.
- تنجح اختبارات Backend وFrontend وبناء Vite في GitHub Actions.
