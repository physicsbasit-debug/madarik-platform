# Phase 4-B1: Real AI Translation Acceptance

## الهدف

إغلاق مسار الترجمة الحقيقية دون السماح للترجمة المحلية الاحتياطية أو النص المختلط عربيًا/إنجليزيًا بالحصول على قبول نهائي.

## نطاق المرحلة

1. إبقاء مفاتيح المزود في متغيرات البيئة فقط.
2. دعم Gemini عبر `generateContent`، وOpenAI الرسمي عبر Responses API، والمزودات المتوافقة عبر Chat Completions.
3. فحص القاموس والمحتوى العلمي المحمي كما في المراحل السابقة.
4. إضافة حارس جودة عربية يرفض النثر الإنجليزي غير المفسر في وضع العربية.
5. إجراء محاولة تصحيح خارجية واحدة عند مخالفة القاموس أو السلامة العلمية أو جودة العربية.
6. استخدام fallback محلي آمن عند استمرار المخالفة أو فشل المزود.
7. منع أي عنصر fallback أو فشل محفوظ بأمان من الوصول إلى حالة `accepted`، حتى لو اعتمد المعلم بطاقة السؤال يدويًا.

## حارس جودة العربية

يسمح الحارس بالرموز والمتغيرات والوحدات والاختصارات العلمية مثل:

- `V = IR`
- `I = 2 A`
- `5 Ω`
- `CMBR`
- `Fig. 1.1`

ويرفض بقايا النثر الإنجليزي مثل:

```text
فسّر why the current decreases.
```

## الإعداد

انسخ `backend/.env.example` إلى `backend/.env`، ثم أدخل مفتاحًا صالحًا ومعرّف نموذج مدعوم من لوحة المزود. لا تُحفظ مفاتيح حقيقية في Git أو ملفات الاختبار أو السجلات.

## الفحوص

```bash
./RUN_PHASE4_B1_TESTS.sh
```

بعد توفير مفتاح صالح:

```bash
python RUN_PHASE4_B1_LIVE_ACCEPTANCE.py
```

الاختبار الحي لا يطبع المفتاح ولا الاستجابة الخام.

## معيار الإغلاق الإنتاجي

لا تُغلق المرحلة نهائيًا بمجرد نجاح الاختبارات الوهمية. يلزم:

1. ترجمة ورقة Cambridge الكاملة بمزود فعلي.
2. عدم بقاء نثر إنجليزي غير مفسر في وضع العربية.
3. سلامة جميع القيم والرموز والوحدات والدرجات.
4. وصول تقرير ترجمة الورقة إلى `accepted` بعد مراجعة المعلم.
5. بقاء أي تشغيل fallback في `needs_review`.
