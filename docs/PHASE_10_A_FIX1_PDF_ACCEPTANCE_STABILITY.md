# Phase 10-A Fix 1: PDF Acceptance Stability

## سبب الدفعة

أظهرت بوابة GitHub Actions أربع حالات فشل كان فيها ملف التصدير النهائي
`needs_review` بدل `accepted`. جميعها تمر عبر فحص PDF، بينما بقية الاختبارات
نجحت.

كان الفحص يعتمد على افتراضين هشين:

1. أن قارئ PDF يعيد عنوان السؤال بصيغة سطرية ثابتة مثل `1 [2]`.
2. أن صور الأسئلة موجودة دائمًا كـ Image XObject مباشرة في موارد الصفحة.

صيغة PDF لا تضمن أيًا من الافتراضين. قد يفصل قارئ النص الرقم عن الدرجة أو يعكس
ترتيبهما بسبب RTL، وقد يضع مولد PDF الصورة داخل Form XObject. كما تظهر أقنعة
الشفافية كصور داخلية، لكنها ليست مرفقات مستقلة.

## الإصلاح

- تطبيع أرقام العربية والفارسية وعلامات اتجاه النص.
- التعرف على توقيع رقم السؤال والدرجة في الاتجاهين وعبر سطرين متجاورين.
- استخدام رقم العرض الفعلي الذي يستعمله مولد التصدير.
- اجتياز Form XObjects بصورة تكرارية.
- استبعاد ImageMask وSMask من عدد المرفقات المرئية.
- تثبيت `reportlab==4.4.9` لتوحيد بيئة الاختبار مع النسخة المجتازة محليًا.

## اختبارات الانحدار

أضيفت اختبارات مستقلة للحالات التالية:

- عنوان RTL بأرقام عربية هندية.
- ترتيب معكوس `[3] 2`.
- انقسام الرقم والدرجة إلى سطرين.
- صورة داخل Form XObject.
- صورة لها Soft Mask معروضة داخل موارد الصفحة.

## معيار القبول

يجب أن تمر:

- اختبارات قبول التصدير المركزة.
- اختبارات الرحلة الكاملة.
- اختبارات Backend كاملة.
- Python compileall.
- بوابة Release Audit.
- GitHub Actions بما فيها Frontend install/lint/build.
