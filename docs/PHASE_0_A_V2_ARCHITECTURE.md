# Phase 0-A: V2 Architecture

## المبدأ العام

تُبنى V2 كتوسعة للنواة الحالية، لا كإعادة بناء من الصفر.

## الطبقات

### 1. طبقة المصادر

- ملفات الجهاز
- Google Drive
- OneDrive لاحقًا
- وثائق المناهج
- الاختبارات الأصلية
- ملفات المشاريع السابقة

### 2. طبقة الاستخراج والمعالجة

- PDF text extraction
- OCR
- استخراج الأسئلة والأجزاء
- استخراج الدرجات
- اكتشاف الرسوم والجداول
- اكتشاف الفقد والتعارض

### 3. طبقة المعرفة العلمية

- الصفوف 1–12
- المواد العلمية
- الوحدات
- الدروس
- نواتج التعلم
- المصطلحات العلمية المعتمدة
- ربط السؤال بالمصدر والمنهج

### 4. طبقة الذكاء الاصطناعي

- الترجمة العلمية
- تصنيف السؤال
- تقدير الصعوبة
- توليد إجابة نموذجية
- توليد أنشطة متمايزة
- توليد أو إعادة بناء رسوم علمية

### 5. طبقة الاعتماد والجودة

- التحقق من اكتمال الأسئلة
- التحقق من الدرجات
- التحقق من الرسوم
- كشف التكرار
- نسبة الثقة
- اعتماد المعلم
- سجل التغييرات

### 6. طبقة الإنتاج

- بنك الأسئلة
- مولد الاختبارات
- جدول المواصفات
- الأنشطة المتمايزة
- Word
- PDF
- نسخ الطالب والمعلم

## قواعد المعمارية

- كل سؤال يحتفظ بالمصدر الأصلي ورقم الصفحة.
- لا يُضاف سؤال إلى البنك قبل الاعتماد.
- لا تُعد الترجمة ناجحة إذا فُقد سؤال أو درجة أو رسم حرج.
- التصنيف الآلي اقتراح قابل للتعديل، وليس حكمًا نهائيًا.
- التكامل السحابي يبقى منفصلًا عن منطق السؤال والبنك.
- إعادة بناء الرسوم وحدة مستقلة ولها fallback آمن.
